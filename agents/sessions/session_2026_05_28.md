# Session Summary — 28/05/2026

**Dự án**: Transport Daily Report (Hệ thống Báo cáo Telegram Tự động & Pilot)

Trong session hôm nay, chúng ta đã xây dựng thành công luồng Báo cáo Tự động hằng ngày bắn thẳng qua Telegram, bao gồm cả link HTML lẫn Hình ảnh thiết kế riêng cho các kho/siêu thị chạy Pilot.

Dưới đây là tóm tắt toàn bộ công việc đã hoàn thiện:

## 1. Hệ thống Bot Auto-Scheduler Telegram
- Viết script `auto_scheduler.py` chạy ngầm, tự động check giờ để sinh báo cáo và gửi Telegram theo lịch cố định:
  - **09:00**: KRC (ngày D) và KSL-Tối (ngày D-1).
  - **15:00**: KSL-Sáng (ngày D).
  - **16:30**: ĐÔNG và MÁT (ngày D).
- **Catch-up Logic thông minh**: Nếu người dùng mở máy trễ (ví dụ 9h30 mới bật), bot sẽ phát hiện ca 09:00 chưa gửi và lập tức tự động chạy bù để đảm bảo không miss bất kỳ báo cáo nào. 
- State lưu trữ chống spam trùng lặp (`telegram_schedule_state.json`).

## 2. Nâng cấp Format Báo Cáo (HTML & Hình Ảnh)
- **Hình ảnh Pilot sắc nét**: Chuyển đổi từ chụp web sang vẽ bằng thư viện `PIL` để xuất ra hình ảnh cực kì rõ nét (dark theme), hiện full Trip ID (không cắt đuôi) với tiếng Việt đầy đủ dấu (Đủ / Thiếu X / Dư Y).
- Hình ảnh Pilot này được set rule: Chỉ generate hình riêng cho 2 siêu thị **HTP & SCV** ở luồng báo cáo lúc 09:00 (KRC và KSL-Tối). Các ca chiều tối chỉ cần gửi link HTML.
- **HTML Flat Table**: Thiết kế lại trang báo cáo HTML từ dạng gộp (group by trip) sang bảng phẳng (flat table) trực quan, có thêm bộ lọc (Search) và summary tự động deploy lên Github Pages.

## 3. Hoàn thiện Logic Data Pipeline 
- **Phân tách ca KSL chuẩn xác**: Đổi logic phân chia KSL-Sáng/KSL-Tối từ việc tính dựa trên giờ tới thực tế (gây nhiễu) sang dùng trực tiếp khóa `DRY_sang` / `DRY_toi` từ file kế hoạch `compose_state`.
- **Merge data NSO**: Chuẩn hóa các dòng định dạng lạ của siêu thị NSO (ví dụ `A154 xe 1`, `A154 xe 2`) về chung 1 mã cửa hàng để map trip mượt mà, không sinh ra dòng thừa.
- **Lọc sạch Trip Hủy (Canceled)**: Cập nhật luồng fetch data từ ClickHouse (`fetch_db_realtime.py`), bít hoàn toàn các chuyến xe bị hủy (`t_status = 4`) ở cả 2 hàm query (detail và tracking), đảm bảo các chuyến hủy bay màu 100% khỏi báo cáo.
